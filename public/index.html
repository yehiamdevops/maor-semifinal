Action Name URL Call Request Method action details
Create User http://host:port/users POST Creating user or admin in the database
User login http://host:port/users/login POST log in to a user or admin account after account created
Show all toys http://host:port/toys?perPage=Number&&page=Number&&orderBy=String&&reverse=String GET perPage=default to 10 toys per page
page = default to show first 10 toys(1)
orderBy = default to sort by db entry order could change to be sorted by any other toy property
reverse = default to no reverse(1) prompt yes to reverse (reverse=yes)
Search toys http://host:port/toys/search?s=String(name or info) GET s = can find toys by their toy name or the info about them dont use it for regular calls
perPage=default to 10 toys per page
page = default to show first 10 toys(1)
Search toys by their category http://host:port/toys/category/some-category?perPage=Number&&page=Number GET pass a toy category you want to display after the category route
perPage=default to 10 toys per page
page = default to show first 10 toys(1)
Display toys by a price range http://host:port/toys/prices?min=Number&&max=Number&&perPage=12&&page=2 GET min = default to 10
max = default to 40
display toys that their price between the min and max perPage=default to 10 toys per page
page = default to show first 10 toys(1)
Display a toy by his id http://host:port/toys/single/some-toy-id GET returns toy details based on the given id.
Number of toys http://host:port/toys/count GET returns the number of toys in the toys collection
Create a toy http://host:port/toys/ POST require a logged account with a valid token
create a toy if the body is valid based on the following model:
name: String,
info: String,
category: String,
img_url: String,
price: Number,
user_id: String
Edit a toy http://host:port/toys/some-toy-id PUT require a logged account with a valid token or an admin token
edit a toy if the body is valid based on the following model:
name: String,
info: String,
category: String,
img_url: String,
price: Number,
user_id: String
sending the same body will result no changes
Delete a toy http://host:port/toys/some-toy-id DELETE require a logged account with a valid token or an admin token
deleting a toy based on the given id.
